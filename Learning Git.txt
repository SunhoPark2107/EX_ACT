git 의 구조

로컬 파일 작업 => 내 컴퓨터 파일에 추가.
 
staging 작업 공간 => 서버로 옮기기 전의 현황. : git add . 으로 "현재"디렉토리의 모든 변경 내용을 staging 영역으로 넘기는 것.

중간에 git status 통해서 staging 영역에 어떤 것이 변경되었는지 알 수 있다.

git commit => 서버 저장 대기 영역으로 저장. (ex. 첫번째 작업물 / 두번째 작업물 / 세번째 작업물 ...)
